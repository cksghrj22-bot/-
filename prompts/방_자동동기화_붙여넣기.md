# 🔄 방 자동 동기화 — 아무 코드방에 이 블록만 붙여넣으면 됨

> 목적: 이찬호가 **직접 pull 안 해도** 두 방이 항상 같은 상태가 되게. 방(Claude)이 알아서 pull/push 한다.
> 공유 트렁크 = **`main`** (이름 안 바뀜). 붙여넣는 즉시 이 방은 그 규약을 따른다.

## 🔒 불변식 (이찬호 지시 2026-07-18 — 절대 규칙)
**모든 방·모든 레포의 커밋 개수·HEAD가 항상 똑같이 맞물린다.** 한 방이 커밋하면 즉시 main에 올려
다른 방도 같아지게 한다. "내 방 커밋 수 ≠ 다른 방 커밋 수" 상태로 응답을 끝내지 않는다.
- **매 응답 첫 행동:** `git pull --rebase origin main`
- **매 커밋 직후:** `git pull --rebase origin main && git push origin HEAD:main` (성공까지 반복)
- **응답 끝나기 전 자가검증(무조건):**
  ```
  git fetch origin main -q && [ "$(git rev-parse HEAD)" = "$(git rev-parse origin/main)" ] \
    && echo "✅ 커밋 일치(HEAD=origin/main)" || echo "🔴 어긋남 — pull&push 다시"
  ```
  🔴면 pull --rebase 후 push 다시. ✅ 나올 때까지 응답 안 끝냄.

동기화 규약(무조건, 매번):
1. **작업(대답) 시작 전에 먼저:** `git pull --rebase origin main` — 다른 방이 한 걸 받아온다.
2. **커밋할 때마다:** `git add -A && git commit -m "..." && git pull --rebase origin main && git push origin HEAD:main`
   (push 거부되면 = 다른 방이 먼저 올림 → 다시 pull --rebase 후 push. 성공까지 반복.)
3. **충돌 나면:** 최신(다른 방) 것을 살리고 내 변경을 그 위에 재적용. 코드면 `python3 -m unittest discover -s tests` + `python3 -m shorts.spec` 통과 확인 후 push.
4. **큰 작업 시작/끝에** `knowledge/방_공유_작업로그.md` 맨 위에 한 줄 남기고 push (다른 방이 "지금 뭐 하는지" 알게).
5. 세션 시작 시엔 `CLAUDE.md → prompts/00_원샷_런북.md → knowledge/방_공유_작업로그.md`를 읽어 현재 상태부터 파악.

이 규약을 어기지 말 것. 매 응답의 첫 행동은 `git pull --rebase origin main`이다.
