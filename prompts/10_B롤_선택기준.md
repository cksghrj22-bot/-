# 10. B롤 자동 선택 기준 — 주제 → 배경 클립 매칭 (2026-07-18 이찬호 지시 박제)

> 이찬호: "스캔해서 B롤 스스로 붙이는 시스템을 만들어서 프롬프트로 쌓아놔라."
> **말 기준표(이 문서)는 사람용. 실제 판별·선택은 코드가 한다:**
>
> ```
> python3 -m shorts.broll "대본 텍스트"          # 주제 판별 → 클립 자동 선택 [카테고리·파일ID·시작초]
> python3 -m shorts.broll --script <대본파일>     # 파일에서 읽어 판별
> python3 -m shorts.broll --list                  # 카탈로그 상태(파일ID 채워졌나) 표
> python3 -m shorts.broll "..." --mark <클립id>   # 사용 처리 → 다음엔 덜 쓴 다른 걸 고름(중복 회피)
> ```
>
> **데이터 정본 = `knowledge/broll_catalog.json`** (엔진이 읽음). 사람 원장 = `knowledge/B롤카탈로그.md`.
> `file_id: null` = 아직 미스캔 → **A방이 드라이브(문방구 등) 스캔해서 파일ID를 JSON에 채운다.**
> 엔진은 file_id 있는 클립을 먼저 고르고, 없으면 "A방 스캔 필요"로 표시한다.

## 규칙 (무조건)
1. **주제와 같은 걸 가리키는 장면**을 쓴다. 커트 영상에 마인드 대본 = 어긋남(2026-07-18 실증: 브랜드·애니송에 커트 써서 폐기).
2. **다이나믹(움직이는) 핵심 동작 구간**만 (프롬프트07). 시작 세팅·매무새·정지컷 배제.
3. **1편 = 고유 원본 1개.** 같은 클립을 연달아 여러 편에 쓰지 않는다(반복 인상).
4. 후보가 여럿이면 카탈로그에서 **덜 쓴 것 우선**.

## 쓰는 법 (B방→A방)
- B방: 대본 주제 보고 위 표에서 카테고리 정함 → 카탈로그에서 **파일ID + 다이나믹 시작초** 골라 A방에 넘김.
- A방: 그 파일 다운(드라이브 device auth) → `--broll <파일> --broll-start <초>`로 렌더.
- 대본 헤더 `# B롤:` 줄에 **[카테고리 · 파일ID · 시작초]** 명시 → 다음 세션도 안 헷갈림.
- 카탈로그에 없는 새 촬영이 필요하면 `knowledge/B롤_촬영요청.md`로 이찬호에게.
