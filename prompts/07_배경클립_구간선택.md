# 07. 배경 클립 구간 선택 규약 — 어느 부분을 잘라 쓸까 (2026-07-15 이찬호 디렉션 박제)

> ⛔ **무조건 적용:** 배경 클립을 쓸 때 아무 데나(특히 앞부분) 잘라 쓰지 않는다.
> **핵심 동작이 살아있는 구간**을 골라 `--broll-start`로 잡는다. 렌더 전에 반드시 스캔해서 확인.

## 절대 쓰지 말 것 (버리는 구간)

- **시작부**: 카메라 세팅·매무새 확인·"잘 나오나" 확인·자리 잡기 (준비 동작 = 영상 이상해짐)
- **끝부**: 정리·중단·카메라 끄러 가는 동작
- **멍한/무의미**: 정지·딴짓·시선 흐트러진 구간

## 골라 쓸 것 (핵심 동작)

주제·나레이션과 **같은 것을 가리키는 진짜 동작**:
- 책읽는 영상 → **책장 넘기고 고심하는 중간부** (표지 확인·세팅 앞부분 X)
- 커트 영상 → 실제 가위질·빗질 동작
- 뛰는 영상 → 실제 달리는 동작 (트랙 세팅·폰 만지는 앞부분 X)
- 상담 영상 → 필기·대화하는 순간

## 방법 (렌더 전 필수)

1. 클립을 **스캔**한다: `ffmpeg -ss <t> -i <clip> -frames:v 1 frame.jpg` 로 여러 지점(예: 길이의 20/40/60/80%) 프레임 추출.
2. 프레임을 **눈으로 확인**해 준비/멍한 구간을 배제하고, 동작이 살아있는 시작점을 정한다.
3. 그 지점을 `--broll-start <초>`로 렌더에 넘긴다. (필요하면 그 구간 길이가 나레이션보다 긴지도 확인)
4. 애매하면 후보 2개를 뽑아 시안 비교.

## 다이나믹 구간 우선 (2026-07-17 이찬호 — 아주 중요)

- **영상이 다채롭고 다이나믹한(움직이는) 구간**을 써야 한다. 정지·밋밋한 컷, 특히 시작부 대충 = **사람들이 안 본다.**
- **블랙박스(dim)를 깔았어도** 배경은 반드시 **움직여야** 한다. dim은 톤이고, 시선을 끄는 건 움직임이다. (dim 깔았다고 정지컷 써도 되는 게 아님)
- 같은 원본이라도 **매번 다른·역동적인 구간**을 골라 반복 인상을 피한다.

## 제안 규칙 — B롤을 대본과 같이 제안 (2026-07-17 이찬호, 무조건)

- **영상을 제안하거나 대본을 만들 때는 B롤도 반드시 같이 제안한다.** 이찬호가 그걸 보고 찍어 올린다.
- 각 B롤 제안 = **무엇을(장면) · 어떤 다이나믹 동작 · 몇 초 · 어느 편에 쓰는지**. 시작 세팅·매무새 구간은 빼라고 명시.
- 대본 파일 헤더에 `# B롤:` 줄로 그 편에 필요한 B롤을 적는다 (없는 클립이면 "촬영 요청"으로 표시).
- 있는 클립(커트·러닝·상담·독서)만 계속 돌려쓰면 같은 영상처럼 보인다 → **부족한 B롤은 촬영 요청 리스트로 이찬호에게 전달**(knowledge/B롤_촬영요청.md).

## 왜 (이찬호 지적)

"스캔했다면서 왜 카메라 매무새 확인하는 앞부분을 잘라 쓰냐" — 스캔의 목적은 **버릴 구간과 쓸 구간을 가르는 것**이다. 스캔하고도 앞부분을 쓰면 스캔한 의미가 없다.
"B롤이 없으니 계속 같은 영상이 나온다" — 그래서 **필요한 B롤을 먼저 말해주고**, 다이나믹한 구간을 골라 쓴다.
