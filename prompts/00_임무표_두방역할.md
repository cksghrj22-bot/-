# 00. 임무표 — 두 방 역할 못박기 (2026-07-18 이찬호 지시로 A방 확정)

> 이찬호: "충돌하기 전에 원래 기준 다시 찾고, 방마다 역할 명확히 해서 연동해라."
> **세션 시작 시 이 표를 제일 먼저 읽는다.** 그다음 `knowledge/방_공유_작업로그.md`(지금 상태) → 규약.
> 충돌(같은 걸 두 방이 만듦)이 세 번 났다: B롤 시스템 2번·브랜치 난립. 이 표가 그걸 끝낸다.

## 결론 (구조)
- **방은 둘 유지**(병렬 속도). 대신 역할을 겹치지 않게 못박는다. 정본 브랜치는 **main 하나.**
- **만들기 전에 이 표 + 「이미 있는 것」을 먼저 본다.** 이미 있으면 다시 만들지 않고 그걸 쓴다.

## 누가 뭘 하나 (겹치면 안 됨)

| 영역 | 담당 | 파일/산출물 |
|---|---|---|
| 영상 렌더 (흑백 첫줄기) | **A방** | `shorts/proof.py` · `render.py` · `subtitles.py` · `tts.py` |
| 유튜브 예약·발행 | **A방** | `shorts/upload_youtube.py` (발행은 이찬호 컨펌 후) |
| **B롤 전부** (드라이브 스캔·file_id 찾기·카탈로그 채우기·구간선택·품질검증) | **A방** | `knowledge/broll_catalog.json`(엔진 데이터) · `knowledge/B롤카탈로그.md`(사람원장) · MCP 드라이브 스캔 |
| 정본 규격 검사 | **A방** | `shorts/verify_render.py` · `shorts/spec.py` |
| 「첫줄기_완성」 관리 | **A방** | 드라이브 `_최신_바로보기`·완성폴더 |
| 대본·제목·설명·태그 | **B방** | `content/shorts/**.txt` |
| **B롤은 카테고리만 지정** (`# B롤: [category]`) | **B방** | 대본 헤더에 카테고리 한 단어. 실제 file_id·구간은 A방이 채움 |
| 상황판·구술정리·볼트 | **B방** | 노션 상황판 · 옵시디언 볼트 |

> **B롤 경계(핵심):** B방은 대본에 **카테고리(cut/consult/workout/reading/aerial/scalp)만** 적는다.
> **실제 클립 file_id·시작초·품질 판단·스캔은 전부 A방.** B방이 B롤 찾기/스캔/카탈로그 코드를 만들면 충돌 — 하지 말 것.

## 이미 있는 것 — 다시 만들지 마라 (원래 기준)
- **B롤 품질 기준**(어느 구간 고를지·정지컷 배제·다이나믹·같은 장면 반복 금지) → **`prompts/07_배경클립_구간선택.md`**
- **B롤 주제→카테고리 매칭** → **`prompts/10_B롤_선택기준.md`** + 엔진 **`shorts/broll.py`** (`--script <대본>`)
- **같은 장면 안 쓰기 (1편=고유 원본)** → **`shorts_config.json`의 `source_video_policy`**(코드가 강제) · `prompts/02`(원칙) · `prompts/10` 규칙3
- **B롤 카탈로그 데이터** → `knowledge/broll_catalog.json` (없는 file_id는 A방이 드라이브 스캔으로 채움)
- **드라이브 스캔(폰 자동업로드 등)** → A방이 MCP Google Drive 커넥터로(승인 팝업 없이 읽힘). drive.file(기기인증)은 폰 파일 못 봄.

## 정본 브랜치 = main (하나)
- 두 방 다 **main**에 pull/push. (CLAUDE.md 자동동기화 규약: `git pull --rebase origin main && git push origin HEAD:main`.)
- feature 브랜치(script-factory·session-restart 등) 난립 금지 — 작업은 main에 모은다.
- 흑백(bw) 정본은 유지. warm_film은 GRADES 팔레트로만.

## 충돌 났을 때
1. 최신 main pull → 양쪽 변경 다 살려 merge(둘 다 일어난 일).
2. 같은 기능을 두 방이 만들었으면 **먼저/더 통합된 쪽을 정본**으로, 다른 쪽은 접는다(중복 엔진 금지).
3. 큰 작업은 시작 전에 `방_공유_작업로그.md` 맨 위에 "이 방이 이거 하는 중" 한 줄 → 다른 방이 안 겹치게.
