> 2026-07-16 이찬호 지시(박제): "이렇게 안 되면 저렇게, 저렇게 안 되면 이렇게 가지 말고,

> 규칙이 코드로 강제되는 곳은 "🔒 코드강제" 표시 — 기억 안 해도 코드가 막는다.

## 0. 세션 시작 원샷 (순서 그대로, 딴 길 없음)

```bash
bash assets/fonts/install_fonts.sh      # 교보손글씨 설치 (안 하면 자막 폴백)
python3 -m pipeline check               # 연결 진단. '없음'이면 CLAUDE.md 자가연결 절차로 키 복원
python3 -m unittest discover -s tests   # 코드 무결성 (외부의존 없이 통과해야 정상)
```

- 진단에서 **지식 인덱스 '없음'이면 즉시 재구축** (CLAUDE.md 새 세션 절차 1번).
- 키가 '없음/차단'이면 → 노션 「코드방 상태판 → 세션 연결 정보」에서 복원. 네트워크는 **「대본공장」(전체 허용) 환경**이어야 일레븐랩스 통과.
- 먼저 읽을 정본: 이 문서 → `knowledge/결정사항_대장.md`(이미 정해진 것, 재질문 금지) → `연결지도.md`(연결 현황).

## 1. 소통 규약 — 왕복을 없앤다 (가장 큰 문제였던 것)

이찬호가 지적한 **최대 문제 = 기억 못 함·재질문·"연결했는데 또 해달라"·이미 한 걸 재승인 요청.**
그래서:

1. **이미 정해진 건 다시 묻지 않는다.** `knowledge/결정사항_대장.md`에 있으면 그대로 실행. 없을 때만 판단.
2. **할 수 있는 건 묻지 말고 실행하고 결과만 보고.** 사용자에게 가는 요청은 **딱 3종**뿐:
   ① 발행 승인(yes/no) ② 로그인 필요한 키 발급 ③ 본진 전달용 복붙 한 줄. 여러 개면 모아서 한 번에.
3. **같은 걸 두 번 승인받지 않는다.** 한 번 OK 난 규격/결정은 코드·문서에 박고 재확인 안 함.
4. **"이어서 계속" 한 줄로 재개되게** 작업물은 항상 파일·깃에 남긴다(채팅 기억 0 전제).
5. 커밋·푸시·레퍼런스 관리는 **Claude가 전부 핸들한다.** 이찬호에게 커밋/깃 얘기로 부담 주지 않는다.

## 2. 작업 경로 — 유형별 한 줄기 (분기 없음)

### 2-A. 쇼츠(마인드) 렌더 — 대본→시안→수락
```bash
# TTS 캐시 재사용(workdir 고정) + 렌더 후 규격 7항목 자동검사(🔒 코드강제, FAIL이면 산출 거부)
python3 -m shorts.proof content/shorts/<날짜> --out content/shorts/<날짜>/보이스_시안 \
  --preset style_preset_mind --only NN --broll <클립> --broll-start <초> \
  --bgm <BGM.mp3> --workdir <스크래치>/wkNN
```
- **텍스트(대본) 방향은 이찬호가 정한다. 대본 이찬호 확인 후에만 렌더** (결정사항_대장 워크플로).
- B롤 시작점은 §4 편집자 뇌로 스캔해서 정한다(준비구간 금지).
- 렌더가 PASS로 끝나야 시안이다. **FAIL이면 코드가 산출을 막는다 — 고쳐서 재렌더.**

### 2-B. 대본/카피 — 막 쓰기 금지 (내 머리로 창작 금지)
**착수 전 `knowledge/옵시디언_볼트_지도.md` → 밤작업 10(형님 톤 정답본) → 해당 개념노트를 먼저 읽는다.**
형님이 **방향(키워드/앵글)** 을 주면, 볼트에서 형님이 실제 쓴 문장을 찾아 **그 어휘·논리로** 쓴다.
볼트에 형님 문장이 없는 주제는 **창작 금지 — 방향/구술 요청**(3종 요청). 구술이 오면 볼트에 원문 저장.
그다음: `00_차노화_마스터` 장착 → `차노_보이스`+`키워드_지도` 로드 → 인덱스 검색 →
`05_까다로운_검수` → TTS 전 `06_나레이션_라임` 부호. (결정사항_대장 「글」.)

### 2-C. 교육 도구 — 코드로 만든다
학생 폰 QR 제출(구글폼) + 큰 화면 실시간 HTML + 폼 생성 Apps Script 원클릭. 보관 `content/교육/<날짜_교육명>/`.
이론 선행 금지 — 궁금한 것부터. (결정사항_대장 「교육」.)

## 4. 편집자 판단 뇌 — 40초 쇼츠의 클립 구성 (세계 최고 편집자로서)

> 2026-07-16 이찬호: "40초짜리에 영상이 너무 다양하게 들어가면 시선이 분산된다.
> 그걸 판단하는 뇌를 장착해서 네가 알아서 분류해라."

- **한 편(≈40초) = 클립 1~2개 원칙.** 하나의 원본에서 핵심 동작 한 구간을 길게 쓰는 게 기본.
  주제 전환이 뚜렷할 때만 2개. **3개 이상 컷 전환 금지**(시선 분산 = 저품질).
- **구간 선택은 `prompts/07_배경클립_구간선택`**: 시작 준비구간·끝 정리·멍한 구간 버리고 **핵심 동작 중간부**. 렌더 전 스캔 필수.
- 주제-화면 일치: 나레이션이 가리키는 **진짜 동작**을 고른다 (자기투자→운동 동작, 단골→시술 교감, 책→책 넘기는 중간부).
- 애매하면 후보 2개 스캔 비교 후 하나 선택 — **이찬호에게 고르라고 떠넘기지 않는다**(3종 요청 아님).

## 6. 다신 까먹지 않게 — 이 시스템이 도는 방식

- 🔒 **규격은 코드가 강제**(proof.py 자동검사) — 기억이 아니라 게이트. 검사기·config·`제작규격_정본.md` 3곳은 항상 일치시킨다.
- **삽질은 그때그때 `knowledge/렌더_버그_학습로그.md`에 원인→수정으로 박제** — 같은 원인 두 번 안 겪는다.
- **결정은 `knowledge/결정사항_대장.md`, 현황은 `연결지도.md`.** 바뀔 때만 갱신.
- 방이 리뉴얼돼도(채팅 기억 0) **이 문서 + 코드 게이트 + 학습로그**만 읽으면 한 번에 재현된다. 그게 이 런북의 목적.
