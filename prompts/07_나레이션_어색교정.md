# 07 나레이션 어색 교정 프롬프트 (2026-07-23 이찬호 "왜 어색하지 분석해서 박제")

TTS 문장이 어색할 때 **원인 진단 → 수정 → 검사** 순서. 케이스 1호: "같이 운동을 했어요".

## 🔍 진단 프롬프트 (왜 어색한가 — 3원인 체크)
어색한 줄을 넣고 아래를 순서대로 점검한다:
1. **구개음화·연음 미반영?** — 표기와 소리가 다른 말인가. (예: 같이=[가치], 굳이=[구지], 해돋이=[해도지], 붙여=[부처]) → 일레븐랩스가 표기대로 [갇이]로 읽으면 딱딱·어색. **표기 그대로면 십중팔구 이것.**
2. **세그먼트 경계 억양 리셋?** — 문맥 스티칭은 줄마다 따로 합성→이어붙임. 줄 첫머리에서 피치가 리셋돼 "새 문장처럼 툭" 시작. 짧은 줄일수록 심함.
3. **과도한 쉼(쉼표)로 토막?** — 한 호흡이어야 할 말을 쉼표로 끊어 staccato. (예: "그래서 오늘은, / 같이 / 운동을 했어요" = 3토막 끊김.)

## 🛠 수정 프롬프트 (3원인 대응)
1. **발음 리스펠(synth 전용)** — synth 텍스트만 소리나는 대로, **자막은 원문 유지.** `shorts/tts.py` SYNTH_FIXES에 추가.
   - 같이 → 가치 (박제 2026-07-23)
2. **쉼표 정리** — 한 호흡 문장의 내부 쉼표 제거. "그래서 오늘은, 같이…" → "그래서 오늘은 같이…"
3. **문맥 늘리기** — 짧은 줄은 앞 줄과 한 세그로 합쳐 억양 리셋 방지(문맥 스티칭이 앞뒤를 함께 읽게).

## 케이스 1호 결론 (검사 대기)
- 문제줄: "그래서 오늘은, 같이 운동을 했어요."
- 원인: ①"같이"=[가치] 미반영 ②쉼표로 뒷부분 툭 시작.
- 후보: **A**(현재) / **B** 리스펠+쉼표제거 "그래서 오늘은 가치 운동을 했어요"(자막은 같이) / **C** 쉼 위치이동 "그래서, 오늘은 가치 운동을 했어요".
- 샘플: `어색분석_ABC.m4a` (A→B→C). **이찬호 검사 후 SYNTH_FIXES·대본 확정.**
