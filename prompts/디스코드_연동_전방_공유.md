## A·C방이 켜는 법 (세션 시작 시 1회)
코드는 이미 레포에 있음. **secrets만 채우면 됨**(gitignore라 커밋 안 됨 → 노션에서 가져옴):
1. 노션 「📱 코드방 상태판」(39d93478-e92b-8196-b20f-d6741e4dafe3) 「🔑 세션 연결 정보」 → 디스코드 섹션에서
   - 봇토큰 → `secrets/discord_bot_token.txt`
   - 채널ID(`1518460128968572958`) → `secrets/discord_channel_id.txt`
   - 웹훅URL → `secrets/discord_webhook.txt`
2. 검증: `python3 -c "from shorts.discord_notify import send; print(send('C방 연동 테스트'))"` → 204면 OK
3. 수신 테스트: `python3 -m shorts.discord_listen once`

⚠️ 같은 봇토큰을 여러 방이 **동시 폴링**하면 메시지 중복수신 가능. 수신(폴링)은 **한 방만**(기본 B방) 담당하고, 나머지 방은 **발신만** 쓰는 걸 권장. 발신은 여러 방이 써도 무방(웹훅).

## 밀리는 일 = 디스코드로 위임 (Creator OS/본진에)
내가(코드방) 밀릴 것 같은 **자질구레한 일**은 `discord_notify.send()`로 Creator OS(본진)에 바로 넘긴다:
```python
from shorts.discord_notify import send
send("[코드방→CreatorOS] 위임: <할 일 한 줄>. 결과는 #assistant에 남겨줘.")
```
- 위임 대상 예: 긴 파일 드라이브 업로드, 반복 렌더 큐, 단순 조회/정리 등 본진이 더 빠른 일.
- 발행·외부액션은 위임 금지(형님 승인 후에만).
- 위임했으면 상황판·작업로그에 "디스코드 위임: <일>" 한 줄 남긴다.
