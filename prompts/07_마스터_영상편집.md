## 6. 글자 깨짐 방지 (shorts/textsafe.py — 2026-07-23 실사고)

- **원인**: 손글씨 폰트가 '공백'을 □ 박스로 렌더(cmap엔 있다고 나오나 실제 깨짐).
- **강제**: 렌더 전 `textsafe.pick_font(text, [손글씨, 나눔])`으로 **실제 렌더 기반**(`broken_chars`) 검증 → 깨지는 폰트 자동 회피. `fit()`으로 폭 초과 자동 줄바꿈·자동 축소. **화면 밖으로 안 잘리고 두부 안 남긴다.**
- **자막 위치도 검증(2026-07-23 실사고)**: ASS `[V4+ Styles]` 꼬리값은 **정확히 9개**(Bold,BorderStyle,Outline,Shadow,**Alignment**,MarginL,MarginR,MarginV,Encoding). 하나 빠지면 libass가 밀려 읽어 자막이 **우상단**으로 튄다(하단중앙=Alignment **2**). **큰 렌더 전 단색배경 1프레임으로 위치 QC** 필수. 상세: `knowledge/렌더_버그_학습로그.md` #9.

## 7. 롱폼 규칙 (박제)

- **원본(DJI·인스타360 등)이 메인 화면.** 인터뷰/발화는 나레이션·자막.
- **하드컷 최소 → 크로스페이드**(xfade+acrossfade). 무음 타이틀카드로 말 끊지 않기(상단 밴드 오버레이).
- BGM 언더(5~14%) + 현장음 유지. 6분 억지로 안 채움 — **내용이 끝나면 끝.**
- **원본 발화가 있으면 그 말을 다 담는다**(텍스트 지어 얹지 않음). 러닝처럼 야외 360캠은 **바람제거 후 STT**(shorts/stt.py, highpass+afftdn) → 형 실제 말 자막.
- **롱폼 자막 스타일(2026-07-23 이찬호 지시 박제)**: **좌측하단** 정렬 · **바(bar)자막**(반투명 검정 박스, 윤곽선/스트로크 없음) · **자간 넓게**(`\fsp4`) · 프리텐드체. ASS: `Alignment 1, BorderStyle 3, OutlineColour 반투명(&H64000000), Outline(박스패딩), Shadow 0`. 쇼츠 정중앙 규격과 다름 — 롱폼은 이 좌하단 바.

## 7-3. 롱폼 편집자 자가 체크리스트 (2026-07-23 박제 — "형은 메시지만, 순서·현장감은 네 몫")
> 이찬호: 순서·문장경계·소리 같은 건 내가 알려주면 안 되는 것. 렌더 전 편집자 뇌로 반드시 자가통과.
1. **시간(=지리) 순서**: 클립 파일명 타임스탬프 = 실제 촬영/이동 순서. **주제로 묶지 말고 시간순 배열이 기본.** 돌아오는 길(오르막 등)이 출발 앞에 오면 안 됨. 비선형(훅먼저)은 **의도했을 때만**, 그렇게 짰으면 형께 "훅 먼저 구성" 명시.
2. **문장 경계 컷**: 발화 시작/끝은 **문장 시작·끝**에서. 말 중간에서 잘라 넣지 않기(특히 인트로 첫 발화). STT 문장 타임스탬프로 경계 확인.
3. **인트로**: 짧게(타이틀 3.4s + 콜드오픈 풍경 ≤6s). 길고 애매한 인트로 금지.
4. **현장음**: 야외 액션캠은 **원래 현장음(바람 포함) 살림**. STT용 강한 디노이즈(afftdn)를 최종 영상 오디오에 걸면 먹먹·비현실 → 자막 입힌 뒤엔 `loudnorm`만(디노이즈 빼기). (2026-07-23 이찬호: "바람 좀 있어도 됨, 답답해")
5. 렌더 후 **첫프레임·인트로·이음매·엔딩** 프레임/소리 자가확인 후 제출.

## 8-3. 검사 제출 워크플로 (2026-07-23 이찬호 지시 — 무조건)
- **영상 검사 맡기기 전 순서**: ①**글(나레이션 텍스트) 먼저** 형께 보여주고 confirm ②일레븐랩스 **띄어쓰기 읽기·어색한 부분** 자가검사 ③어색하면 프롬프트/텍스트 다시 돌려 **2회 이상 필터링 통과** ④그 다음에야 형께 영상 제출. "이렇게 텍스트 주면 수정 쉽네"(이찬호).
- **자가검사 2회**: 렌더 후 스스로 2번 확인(오디오 끝 안 잘림·자막 싱크·발음·레이아웃). 안 괜찮으면 다시 돌려서 통과 후 제출.
- ⭐⭐**제출 전 자동 게이트 필수 — 통과 못 하면 형께 절대 안 보냄** (2026-07-24 이찬호 "필터링 2회 안 거치고 실수 반복"). `scratchpad/miyong3/qc_gate.py <파일들>`로 아래 4개 자동검사 → **ALL PASS만 제출**. 실패면 고쳐서 재검(보고 금지). 게이트 통과 후 **2차=시각검사**(전구간 프레임 스캔: 글리치·반복). 두 관문 다 통과해야 SendUserFile/업로드.
- ⭐⭐**게이트(자동QC)는 화면 "내용"의 이질감을 못 잡는다 — 시각검사를 절대 생략 금지** (2026-07-24 이찬호 "체크 한번 더 했지?" → 두피에 **폰 홈화면 클립(scope4)** 섞여 있던 것 발각). 게이트는 소리·프레임수·길이만 검증. **주제와 안 맞는 클립·엉뚱한 화면(폰 홈스크린·룩북·타 편 소스)은 오직 눈으로만 잡힌다.** 재전송·재렌더마다 `ffmpeg tile`로 12프레임 시트 떠서 **직접 보고** 이질 클립 0 확인 후 제출. "게이트만 돌리고 보고" = 형 규칙 위반. 불량 소스는 발견 즉시 `_BAD_*`로 리네임해 재사용 차단(scope4=폰홈화면 폐기 완료).
  1. **디코드 무결성**(ffmpeg -f null 에러 0)
  2. **길이** = 나레이션+아웃트로
  3. ⭐**비디오 프레임수 = 길이 일치**(2026-07-24 펌 "중간 깨짐" 사고: 짧은 클립 offset+세그길이>클립길이 → xfade truncate로 **영상 7.5초에서 프리즈, 오디오만 34초**. 컨테이너 길이·디코드·소리 다 멀쩡해 안 잡혔음). `ffprobe -count_frames`로 frames ≥ dur×29 확인. **render_lib은 세그에 `-stream_loop -1` 넣어 짧은 클립도 세그 꽉 채움**(재발 방지).
  4. **소리 레벨**: mean_volume ≥ -20dB **AND** 끝 1.5초(BGM only) ≥ -34dB(BGM 살아있음). "소리 안 들려"(amix normalize로 반토막)·"BGM 안 나옴"(최종 loudnorm이 나레이션에 맞춰 BGM 깔아뭉갬) 둘 다 여기서 차단. render_lib 믹스=나레이션 speechnorm + BGM loudnorm -24 따로 정규화 + amix normalize=0 + alimiter(최종 loudnorm 금지).
- ⭐**반복 금지 (2026-07-24 이찬호 재지시 "전체 사이클 한 번 반복도 보기 싫다")**: 클립을 그냥 순환시키지 마라. 순서는 다음 위계로 만든다.
  1. **긴 원본은 시간축 멀리 떨어진 서로 다른 구간(offset)을 최대한 뽑아 쓴다** — "영상 길이를 효율적으로." 같은 클립이라도 관찰/거품/헹굼/마사지처럼 **다른 동작 구간**이면 새 장면. 재등장 시 `hflip`/줌 변형(render_lib 세그 3번째 인자 "h"/"z")으로 더 차별화.
  2. **그래도 반복될 것 같으면(=중복 회피할 footage가 없으면) 자막-싱크 텍스트 카드로 새 장면을 만든다** (이찬호 지시·IMG_2060 레퍼런스). render_lib `("TXT","표시문구")` 세그 = **검정 배경 + 대형 중앙 텍스트**(하단 자막과 싱크). 훅·핵심정의·펀치·마무리 줄에 배치하면 반복도 없애고 시각 반전(중간 훅)까지 얻는다. 미용(컬러) 편 속 검정 카드 = 강한 반전.
  3. **손 배치 우선**: 클립 수가 적거나 짧으면 `alloc` 자동배정보다 18~23세그를 **손으로 (clip,offset,변형) 직접 지정**해 인접 중복 0 + offset 전부 distinct를 보장한다(build_final.py 방식). BGM 볼륨은 `render(bgm_lufs=)`로 편별 조절(예: 멜로디 강한 곡 -30).

## 8-3c. 육성 마인드 롱폼(>50s) 렌더 — 버그#13 우회 (2026-07-24 박제)
- **문제**: 형님 육성 흑백 마인드 편이 45~55s(스토리 아크)라 ①`style_preset_mind` 게이트의 **길이 40~50s 제한**에 걸려 "산출 거부" 뜸(그러나 mp4는 이미 생성됨) ②형 육성 TTS가 조용함(mean ≈ -28dB)+짧은 BGM(48s)이 55s 영상서 꼬리 무음 → qc_gate 2개 FAIL(소리작음·꼬리BGM).
- **✅정식 우회(2026-07-24 코드화) = `--longform` 플래그**: `python3 -m shorts.proof … --preset style_preset_mind --longform` 한 번이면 (a)loudnorm −16 자동 후처리 (b)길이 상한 90초로 완화 → **수동 후처리·재렌더 불필요, 게이트 통과**. 이후 `qc_gate` + 전구간 시각검사만.
- (구 수동법, 참고): proof는 게이트 FAIL이어도 파일을 남기므로 그 파일에 `ffmpeg -i in -c:v copy -af "loudnorm=I=-16:TP=-1.5:LRA=11" -c:a aac -b:a 192k out` 걸어도 동일 결과.
- 길이 40~50s 제한은 **punchy 쇼츠 기준** — 형 육성 스토리(러닝인생 톤)는 `--longform`으로 90초까지 허용(내용·아크 보존 우선).

## 8-6b. ⛔"안 된다" 금지 — 우회 끝까지 (2026-07-23 이찬호 강한 지적, 박제)
- **"접근 안 됨 / 디스크 초과 / 용량 커서 불가" 같은 보고 전에 우회를 끝까지 시도한다.** 대부분 우회 가능하다.
- 실증(반성): 창엽 35GB 롱폼을 "디스크 초과·형 파일 접근 불가"로 **하룻밤 묶어뒀다** → 사실은 ①**직접 익명다운 되는지 테스트**(창엽 폴더는 공유였음) ②**4K 받아 1080p로 낮추고 원본 삭제**(디스크 안에서 순환) ③11.5GB는 **range 분할다운→조각변환→이어붙이기** 로 처음부터 가능했다. 형: "할 수 있는 걸 왜 안 된다고 했다가 이제 와서 된다 그래."
- 규칙: 막혔다고 보고하기 전 **우회 3개 이상 실제 시도**(테스트 다운·프록시화·range·포맷변환·다른 폴더). 그래도 진짜 안 되면 그때만 형께, 시도한 우회를 명시하고 보고.
- 대용량 영상 표준 처리: **4K→`ffmpeg scale=-2:1080` 프록시(crf22 veryfast)→원본 삭제**. 통째로 안 들어가면 **ffmpeg로 URL 직접 스트리밍 변환**(입력=usercontent 다운URL, 4K 저장 X). 검증됨 2026-07-24.
- ⛔⛔**메타버그 = "회로를 못 찾음"** (2026-07-24 이찬호 지적): **능력이 있는데 해결 경로를 스스로 못 찾고, 재명령을 받아야 그제서 찾는 것 자체가 버그다.** 창엽 35GB를 어제 "안 됨"이라 했다가 오늘 시켜서야 뚫은 것 = 이 버그. **강제 습관**: 요청받은 즉시 "이걸 되게 하는 경로가 뭐지?"를 능동 탐색(분해·테스트·우회 나열) → 하나라도 통하면 실행. "안 된다"는 **실제 3개+ 시도 후에만** 나올 수 있는 결론. 형이 "그거 되잖아"라고 다시 말하게 만들면 실패다.

## 11. 골든넷 원본 — 재아 러닝메이트(둘째 줄기) 재현 스펙 (2026-07-23 예약통과 박제)

## 13. 사상 전달 영상 편집 원칙 (2026-07-23 이찬호 박제 — 창엽교육 라인 계기)
> 대상: **전달하는 '생각·사상'이 목적인 영상**(창엽교육, 질문답, 커트사상, 차노의 책장 롱폼 등). 손기술·비주얼보다 **메시지가 1순위**.
- ⭐**말은 끝까지 기다린다.** 컷을 타이트하게 붙이려다 **발화 중간을 끊지 않는다.** 한 문장/한 생각이 **확실히 맺음(마침)을 지은 뒤**에 다음 컷으로 넘어간다.
- **일부러 줄이지 않는다.** 러닝타임 맞추려 말을 자르는 것 금지 — 내용이 다 전달되면 끝, 덜 됐으면 남긴다(§7 "내용 끝나면 끝"의 강화판).
- 컷 전환점 = **문장·생각의 경계**(숨 떨어지는 지점). 말 붙어 있는 구간은 화면만 B롤로 바꾸되 **오디오는 끊지 않는다**.
- 무발화(생각하는 텀·"음…")가 길면 그 **구간만** B롤/점프컷으로 좁힌다. 말 자체는 온전히.
- ⭐**생각 단위(thought-unit)로 편집한다** (2026-07-23 이찬호 추가): 한 생각이 **시작하는 지점~끝나는 지점**을 경계로 삼는다. 마이크(창엽 착용) 오디오라 **파동(진폭)·무음으로 경계가 드러남** → VAD/무음검출로 생각의 시작·끝을 잡되, **그 한 단위 안은 절대 손대지 않고 풀(full)로 통째 유지.** 편집 = 생각 단위들을 **잘린 데 없이 이어 붙여**, "충분히 생각이 전달되는 온전한 단위들의 연속"이 되게 한다.
- 적용: 창엽교육처럼 커트 장면이 거의 없는 소스는 **생각을 줄기**로 편집(말이 주인공, 화면은 받침). §7·§7-3·§8-5와 한 줄기.
