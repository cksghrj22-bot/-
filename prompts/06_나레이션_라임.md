# 06. 나레이션 라임 규약 — 일레븐랩스가 형님처럼 읽게 (2026-07-15 이찬호 디렉션 박제)

> ⛔ **무조건 적용 (예외 없음, 2026-07-15 이찬호 지시):**
> 나레이션에 쓸 텍스트는 — **형님이 준 것이든, 내가 만든 것이든** — TTS(일레븐랩스)에 넘기기 전
> **반드시 이 규약으로 부호를 다시 찍는다.** 이 포맷을 거치지 않은 텍스트는 합성 금지.
> "텍스트를 어떻게 줘야 클론이 안 헷갈리고 이 패턴대로 읽는가"의 정답이 이 문서다.
>
> 대본의 **문장부호가 곧 낭독 지시서**다. 코드는 부호를 조작하지 않고 그대로 읽힌다.
> 그래서 이 규약대로 대본을 쓰면(부호를 정확히 찍으면) 일레븐랩스가 형님 리듬으로 읽는다.
> 자막에도 이 부호가 그대로 보인다 (레퍼런스도 "조금 줘요."처럼 마침표가 보임).

## 부호 = 낭독 지시 (핵심 3개)

| 부호 | 낭독 | 언제 |
|---|---|---|
| **마침표 `.`** | **내려읽고 쉰다** | 한 생각/라임이 **끝나는 줄** |
| **쉼표 `,`** | **살짝 쉰다** | 열거·강조·한 박자 (하나, / 둘, / 그 생각, / 고객은,) |
| **부호 없음** | **다음 줄과 붙여 흐른다** | 문장이 이어질 때 (성의는 늘 하던 대로 → 스타일은…) |

## 규칙 (일레븐랩스 혼동 방지)

1. **라임 끝 줄 = 마침표.** 종결어미(~요/~죠/~다/~까/~습니다)로 끝나면 대개 마침표 → 내려읽고 쉼.
2. **이어지는 줄 = 부호 없이.** 앞 줄과 한 문장이면 앞 줄 끝을 비워 붙여 읽힌다. (예: "성의는 늘 하던 대로" ↵ "스타일은 늘 하던 것의 반대로." — 앞 줄은 무부호로 붙이고, "반대로."에서 내려읽음)
3. **동사/종결어미는 줄 안에 붙여 끝낸다.** 어중간하게 끊으면 일레븐랩스가 헷갈린다.
   - ✗ "늘 같은 머리만" → ✓ "늘 같은 머리만 하게 돼요."
4. **구어체 인용은 구어체 그대로.** "안 어울려요"는 실제 말하듯.
5. **쉼표 남발 금지.** 줄마다 쉼표 찍으면 줄마다 쉬어서 '뚝뚝'. 진짜 쉴 곳에만.
6. **한 줄 = 한 호흡/자막 단위** (≤2줄, 세그먼트 ≤26자). 부호가 그 줄의 끝맺음을 정한다.

## 적용 예 (「단골이 떠나는 이유」 라임)

```
단골 만드는 법, 다들 거꾸로 알아요.      ← 마침표: 내려읽고 쉼
단골이 왜 떠나는지를 보세요.            ← 한 호흡으로(중간 안 띔), 끝 내려읽음
이유는 딱 두 가지예요.
하나, 처음보다 성의가 없어져서.         ← 하나, 살짝 쉼
둘, 새로운 제안을 안 해서.             ← 둘, 살짝 쉼
안 떠나면 그게 곧 단골이에요.          ← 라임 끝: 마침표
성의는 늘 하던 대로                   ← 부호 없음: 다음 줄과 붙여 흐름
스타일은 늘 하던 것의 반대로.          ← "반대로." 내려읽고 라임 종료
"안 어울려요"는 디자이너 생각이죠.      ← 구어체 인용
그 생각, 늘 맞지도 않더라구요.         ← "그 생각," 살짝 쉼
내 생각에 잠기면 늘 같은 머리만 하게 돼요.  ← 동사 붙여 끝냄
그러다 고객은, 조용히 떠납니다.        ← "고객은," 쉬고 → "조용히 떠납니다."
떠나는 이유만 알아도, 단골은 남아요.
```

## 코드 쪽 (이미 반영)

- `shorts/tts.py` build_narration_single: 부호 조작 안 함, 대본 그대로 합성(natural_flow) → 일레븐랩스가 부호대로 쉼.
- 글쓰기 파이프라인에 이 규약을 넣는다: 대본 마무리 단계에서 **부호를 라임에 맞게 찍는다**.
