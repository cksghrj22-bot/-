# 차노화 콘텐츠 엔진 — 프롬프트 라이브러리

한 사람의 지식 자산을 무한 콘텐츠로 바꾸는 프롬프트 세트. AT NOWN에서 실전 운용 중인 시스템의 정본이며,
브랜드 고유명사만 바꾸면 **어떤 전문가에게도 이식 가능한 판매용 패키지**다.

## 구조 (파이프라인 순서대로)

| # | 프롬프트 | 역할 | 입력 → 출력 |
|---|---|---|---|
| 00 | 차노화_마스터 | 페르소나·문체·세계관 고정 | (모든 프롬프트 앞에 붙는 시스템 프롬프트) |
| 01 | 조합엔진 | 키워드×자산×이질개념 → 아이디어 | 키워드 지도 + 씨앗 → 아이디어 N개 (점수화) |
| 02 | 쇼츠_대량생산 | 아이디어 → 촬영 가능한 대본 | 아이디어 목록 → 대본 10~100편 (.txt 파이프라인 형식) |
| 03 | 카드뉴스 | 아이디어 → 캐러셀 | 아이디어 1개 → 6~9컷 카피 + 디자인 지시 |
| 04 | 매거진_블로그 | 원자 → 요리된 깊이 | 쇼츠/카드뉴스 반응작 → 블로그 완성본 |
| 05 | 까다로운_검수 | 품질 게이트 | 초안 → 통과/반려 + 수정 지시 |

## 운용 수칙

1. 항상 00을 시스템 프롬프트로 깔고 01→02/03→05 순서로 돌린다. 04는 반응이 검증된 원자만 승격.
2. 지식 원천은 `knowledge/` 폴더(또는 노션·Obsidian vault)에서 공급한다 — 엔진은 재료 없이 돌리지 않는다.
3. 발행은 반드시 사람이 컨펌한다 (단일 줄기 규약).
4. 다른 브랜드에 이식할 때: 00의 [세계관 상수]와 `knowledge/`의 보이스·키워드 문서만 교체하면 된다.

## 판매 포지셔닝 (참고)

"AI에게 글을 시키는 프롬프트"가 아니라 **"당신의 20년 노하우를 콘텐츠 공장으로 바꾸는 운영 체계"**로 판다.
프롬프트 6장 + 지식 문서 템플릿 2장(보이스/키워드 지도) + 운용 수칙 = 하나의 상품.
