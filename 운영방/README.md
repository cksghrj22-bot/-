# 🏢 앳나운 운영·문제해결 방 (콘텐츠 공장과 분리)

> 이찬호 2026-07-21 지시: 콘텐츠 제작방(쇼츠·블로그·영어자막 등) 말고,
> **일반 운영 문제해결 전용 방**을 따로 둔다. 레포에 커밋된 것 그대로 이어서, **완전 접근허용**으로.

## 이 방의 스코프 (콘텐츠 공장이 아님)
1. **율이 입봉심사 채점판 수거·집계** — 위원 12명 개별 사본/공유 파일에서 점수 모아 종합. 현황판 시트: `율이_입봉심사_수거현황_자동`(드라이브). 현재 2/12 제출(창엽 합격·재아 불합격).
2. **앳나운 교육개편안 (B안 채택 예정)** — 개편 방향 정리·문서화.
3. **선생님별 교육표 → 앳나운 교육시스템 가공** — 각 선생님이 짠 교육표(커리큘럼)를 수집·표준화해서 하나의 앳나운 교육시스템으로 재구성.
4. **일반 운영 문제해결** — 매장 운영·시스템·프로세스 관련 요청.

## 이 방이 지키는 것 (레포 공통 규약 그대로 이어받음)
- **연동 먼저, 그다음 콘텐츠** / **배수관 먼저, 양동이 나르지 말자** (`docs/생각공장_배관_설계도.md`)
- **박제 = 커밋** — 모든 결정·산출은 레포에 커밋(대화 사라져도 남게).
- **표준은 글이 아니라 검사코드로 잠근다** — 규칙 정하면 검증도 함께(`knowledge/결정사항_대장.md` 최상위 원칙).
- **발행·외부액션은 승인 후에만.** 병목·오류는 명령 기다리지 말고 알아서 처리(디렉터).
- 방 자동 동기화: 응답 전 `git pull --rebase origin main`, 커밋마다 pull 후 push.

## 산출물 위치
- 문서·표준화 결과 → 이 `운영방/` 폴더에 커밋.
- 폰 열람용/공유용 → 노션 또는 드라이브(링크 병기).
- 교육시스템 초안 → `운영방/교육시스템/`.
