# 러닝 롱폼 — 예상 문제 사전 점검 & 해결 프롬프트 (2026-07-23)

> 이찬호 지시: "이제까지 고치라 한 오류 + 네가 추론한 문제(버그·글자깨짐·딱딱한 컷·내가 싫어할 편집)를 미리 보고하고, 해결 프롬프트까지 만들어 제시한 뒤 렌더해서 보여줘."

## A. 이미 겪은 오류 → 예방 장치 (박제됨)

| # | 과거 오류 | 예방 |
|---|---|---|
| 1 | **글자 깨짐** — 손글씨 폰트가 공백을 □로 렌더 | `textsafe.pick_font`(실제 렌더 검증)로 깨지는 폰트 자동 회피 → 나눔B. ✅적용 |
| 2 | **엉뚱한 소스** — 헬스영상을 러닝영상으로 착각 | 이번엔 인스타360 러닝 POV 확인 완료. 편집 전 소스 재확인 |
| 3 | **내 텍스트 지어 얹음** — 형 말 대신 내가 쓴 자막 | 이번은 **형 실제 발화(STT)만** 자막. 지어내기 0 |
| 4 | **말 잘라먹음** — 5분에 형 말 다 못 담음 | 001·004·005 발화 **전부** 유지(003 도착 시 삽입) |
| 5 | **딱딱한 하드컷** | 크로스페이드 0.8s + 문장/쉼 경계에서만 컷 |
| 6 | **거친 표현("무시")** | 화면 텍스트 톤 점검 |

## B. 이번 러닝 롱폼에서 추론되는 문제 → 해결

| # | 예상 문제 | 왜 | 해결 |
|---|---|---|---|
| B1 | **자막 오버플로우** | 형 말이 길어 한 줄이 화면 폭 초과 | 실측결과 최장 1115px<1680 안전. +ASS 폭가드 |
| B2 | **자막 오타(STT)** | STT 완벽 아님 | 전사 리뷰 §C. 필러(어·음·그죠) 정리, 명백오류 수정, 불확실은 형께 표기 |
| B3 | **페이싱 드래그** | 무발화 러닝 구간이 길면 지루 | 발화 사이 **1.5초+ 죽은구간(무음) 자동 컷** — 말은 다 살리고 침묵만 줄임 |
| B4 | **오디오 로봇화/펌핑** | 과한 노이즈제거·loudnorm | afftdn nf=-20(중간), loudnorm 1패스, 현장감 유지 |
| B5 | **BGM이 말 방해** | 피아노가 목소리 위 | BGM 5%로 낮춤(말 우선) |
| B6 | **003 누락** | 업로드 오류로 미도착 | 지금 4개로 만들고, 003 오면 002 뒤 삽입해 v3 |
| B7 | **길이 8분+** | 발화 분량 | 형 지시 "6분 억지 안 채움·말 다 담아" → 분량대로 OK |
| B8 | **제목카드 거슬림** | 형이 "텍스트 얹지 마" | 제목은 표준이라 4.5초 짧게 유지 — 빼길 원하면 뺌 |

## C. 자막 오타 — 실제 발견 (형 확인 필요)

- ⚠️ **005: "오늘 하루 사비안 별로 세게 찍자"** — "사비안"은 STT 오인식으로 보임. 문맥(스티브 잡스 '점') 상 "**희미하게 점 찍지 말고 오늘 하루[의 점]을 별처럼 세게 찍자**" 취지. → **형 실제 워딩 확인** 부탁(임시로 "사비안" 빼고 자연스럽게 다듬음).
- 필러 정리 대상: 문장 사이 단독 "어.", "음.", "그죠?", "어?" → 자막에서 뺌(말 흐름은 유지).
- 그 외 001/004/005 전사는 형 어투 그대로 정확 — 살림.

## D. 해결 프롬프트 (이 편집에 강제 적용하는 체크리스트)

```
[러닝 롱폼 편집 규칙 — 렌더 전 전부 통과]
1. 폰트: textsafe.pick_font로 깨짐 검증(공백 □ 방지). 자막 폭>1680px면 자동 줄바꿈.
2. 자막 = 형 실제 발화만(STT). 지어내기 금지. 필러(어/음/그죠/어?) 단독 줄 제거.
   명백 오인식(사비안 등)은 문맥으로 최소 수정 + 형 확인 목록에 남김.
3. 컷: 하드컷 금지 → 크로스페이드 0.8s. 문장/쉼 경계에서만 자름.
4. 페이싱: 발화 사이 무음 1.5초+ 자동 컷(말은 100% 유지, 침묵만 축소).
5. 오디오: 바람제거 nf=-20(과하지 않게)+loudnorm. 목소리 선명·현장감 유지. BGM 5% 언더.
6. 소스: 러닝 POV(인스타360) 확인. 003 미도착이면 표기하고 도착 시 삽입.
7. 화면: 1920x1080, warm 톤(러닝 생동감). 자막 흰글씨+검은박스 하단.
8. 발행 금지: 드라이브 시안만. 유튜브는 형 yes 후.
```

이 규칙을 반영해 v2를 렌더 → 드라이브 업로드 → 자막 깨짐/오타 프레임 확인 후 보고.
