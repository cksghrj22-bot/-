# 📲 폰 영상 → B롤 수급 프로토콜 (2026-07-18 이찬호 지시)

> 이찬호: "폰에서 가져갈 승인 줄 테니 스캔해서 가져가라. 앞으로 스크린샷·찍은 날짜 알려줄 테니
> 구글드라이브나 iCloud에서 가져가면 좋겠다."

## 접근 경계 (코드방이 실제로 되는 것 / 안 되는 것) — 솔직히
- ✅ **구글드라이브: 됨.** 코드방(MCP Google Drive 커넥터)이 **승인 팝업 없이** 읽고 다운로드한다.
  → **드라이브가 유일한 다리.** 폰 영상이 드라이브에 있기만 하면 바로 가져다 쓴다.
- ❌ **iCloud: 안 됨.** 코드방은 iCloud에 연동돼 있지 않다(공유링크 다운로드도 불안정). iCloud 말고 드라이브로.
- ❌ **폰에서 직접: 안 됨.** "폰에서 바로"는 불가 — 반드시 **폰 → 드라이브**를 거친다.
- ⚠️ 별도 '승인 권한' 발급은 필요 없다. 드라이브 읽기는 이미 열려 있고, **파일이 드라이브에 올라와 있느냐**만 관건.

## 표준 흐름 (앞으로 이대로)
1. **폰 → 드라이브 올라오게 한다.**
   - 자동: 폰 사진 동기화(구글포토/드라이브 백업) ON. ※ 멈춰 있으면("동기화 일시정지") **재개**.
   - 수동: 쓸 영상을 「영상문방구」(공유됨) 또는 `_최신_바로보기`에 업로드.
2. **이찬호가 힌트를 준다:** 스크린샷 + **찍은 날짜/대략 시간**(+ 장소·길이면 더 빠름).
3. **A방이 찾는다:** 드라이브에서 그 날짜·폴더로 검색 → 매칭 파일 확정(썸네일/길이 대조).
4. **A방이 가져다 쓴다:** curl/다운로드 → `broll_catalog.json`에 카테고리 등록 → prompts/07대로
   다이나믹 구간 골라 렌더. 같은 클립 재사용 금지(1편=고유 원본).

## 힌트가 정확할수록 빠르다
- 필수: **찍은 날짜**(대략도 OK). 있으면 좋음: 시간대·장소(헬스장/매장)·영상 길이·주제(운동/커트/상담).
- 스크린샷의 파일명(IMG_xxxx / VID_xxxx)이 보이면 그걸로 바로 매칭.

## 유튜브 채널 스캔(별건, 검토 중)
- 이찬호 채널을 B롤 라이브러리로 쓰는 아이디어는 유효하나, 유튜브에서 파일 직접 다운로드는
  별도 도구가 필요하고 막힐 수 있어 **테스트 후 확정**. 원본이 결국 드라이브에 있으면(자막·제목 없는
  원본) 드라이브 쪽이 더 깨끗하다.
