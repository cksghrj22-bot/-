# 🏗️ 지식 파이프라인 구조 — "이찬호 한 마디 → 완성" (2026-07-17 설계)

> 이찬호: "일하는 게 최소화되도록 구조를 짜라. 얼마나 걸려도 상관없다. 구조가 가장 중요하다. 무조건 되게 하라."
> 목표: 이찬호는 **3동작**만 — ①클립 던지기 ②한 마디 ③수락. 나머지는 코드방이 전부.

## 흐름 한 장 (INPUT → 명령 → 처리 → OUTPUT)

```
[촬영/소스]                     [이찬호 한 마디]              [코드방 자동]                 [OUTPUT]
폰 촬영 ─┐                                              ┌ 클립 받기(공유 상속)          ┌ 시안 보고 → 수락
        ├→ 드라이브              "인트레이 새 영상 +       ├ 핵심 동작 구간 스캔    →     ├→ 드라이브 _최신_바로보기(덮어쓰기)
본진 CreatorOS 원본 ─┘  「코드방_B롤_인트레이」   <텍스트/방향>"            ├ 첫 줄기 스타일 렌더         └ (발행 승인) 유튜브/인스타
                        (공유됨)                또는 "IMG_xxxx + 텍스트"   └ 규격 7항목 자동검사(FAIL=거부)      └→ 본진 픽업(브리지=드라이브)
```

## 입력(INPUT) — 클립이 코드방에 닿는 길
- ⭐ **폰 → 드라이브 「코드방_B롤_인트레이」**(ID `1MBvVanqFgvBjk7hS2wjOaYN6oiaoBtlO`, 공유 anyone-reader).
  여기 던지면 공유 상속돼 코드방이 자동 접근. **폰에서 이 폴더 별표 한 번**이 유일한 1회 설정.
- 본진 CreatorOS 원본 → 드라이브 「앳나운_영상」 브리지로 올려두면 코드방이 받음.
- 개별 클립을 딴 폴더에 올렸으면 그 파일만 "링크 있는 사람 뷰어" 공유하면 동일.

## 명령(COMMAND) — 이찬호가 하는 한 마디
- "인트레이 **새 영상** + <텍스트 또는 방향>" → 코드방이 인트레이 최신 클립을 씀.
- "**IMG_xxxx** + <텍스트>" → 파일명 지정.
- **구술**이면 → 코드방이 볼트에 원문 저장 후 대본화(볼트-우선 규칙).

## 처리(PROCESS) — 코드방이 자동으로
1. 클립 다운로드(공유 URL) → 2. 구간 스캔(준비구간 버리고 가위질 등 핵심동작) →
3. **첫 줄기 편집 스타일**(`knowledge/첫줄기_편집스타일_정본.md`)로 렌더 →
4. **규격 7항목 자동검사**(proof.py 강제, FAIL이면 산출 거부·자동 재조정).

## 출력(OUTPUT)
- 시안을 이찬호에게 보고 → **수락** → 드라이브 `_최신_바로보기`에 덮어쓰기(같은 링크·중복0).
- 발행은 **승인 후에만**(유튜브 일부공개→공개 / 인스타). 발행 시간 9·16·23시.
- **본진/CreatorOS 연동 = 드라이브 브리지.** 본진이 `_최신_바로보기`·「앳나운_영상」을 읽어 픽업.
  (본진이 CreatorOS 전용 수신 폴더를 정하면 그 폴더로 자동 업로드하도록 확장 가능.)

## 정직한 경계 (무엇이 되고 안 되는지)
- ✅ **되는 것(코드방)**: 드라이브 읽기·업로드(앱 OAuth), 렌더, 규격검사, 깃 커밋, 유튜브 업로드(키 있음), 노션·Gmail·볼트 읽기.
- ❌ **여기서 직접 안 되는 것**: iCloud 연동(커넥터 없음 — 폰 드라이브 앱 경유로 우회), CreatorOS 직접 실행(본진 Mac 전담), **여기서 예약 트리거 생성**(단일 줄기 규약 — 예약·발신은 본진 유일).
- 그래서 **iCloud/CreatorOS는 드라이브를 다리로** 쓴다. 이게 "무조건 되게" 하는 현실 경로.
