# 📥 유튜브 영상 다운로드 루트 (2026-07-24 이찬호 "무조건 뚫어라")

> 목적: 유튜브에 올린 **내 롱폼**을 바로 꺼내 B롤/재활용 → 콘텐츠 넓이 폭발. (내 영상 = 내 소유, 합법)

## 결론: 도구는 yt-dlp, 루트 작동 확인됨
- 표준 도구 **yt-dlp** (이 환경에 설치돼 있음, 2026.7.4). 유튜브 URL로 내 영상 다운.
- 기본 명령:
  ```
  yt-dlp -f "bestvideo*+bestaudio/best" -o "%(title)s.%(ext)s" "<유튜브_URL>"
  ```
  - 비공개/일부공개 영상: `--cookies-from-browser chrome` 또는 `--cookies cookies.txt`(로그인 세션) 추가.
  - 특정 구간만: `--download-sections "*00:10-00:40"`.

## ⛔ 코드방(클라우드) 세션의 한계 (2026-07-24 실측)
- **youtube.com API/메타데이터는 도달 O** (형식 리스트·제목 다 읽힘).
- **실제 영상 데이터 호스트 `googlevideo.com` = 403 Forbidden** (이 세션 egress 정책 차단). → 코드방에서 직접 다운로드 **불가**. 조직 정책이라 우회 금지.

## ✅ 되는 경로 (택1)
1. ⭐**본진(맥스튜디오, 네트워크 전체 허용)에서 yt-dlp 실행** → 받은 영상 구글드라이브 「앳나운_영상」에 업로드 → **코드방이 curl(usercontent)로 끌어 씀**(기존 실사 B롤 브리지 그대로). 가장 확실.
2. **코드방을 네트워크 「전체 허용」 환경으로 열면** 여기서도 바로 다운로드 됨(대본공장 환경처럼 egress 개방).
3. **자동화**: 본진에 yt-dlp 원클릭 스크립트("URL 붙여넣으면 받아서 드라이브 업로드") 두면, 코드방에 URL만 주면 됨.

## 실무 워크플로 (권장)
- 형이 재활용하고 싶은 롱폼 URL(들)을 준다 → 본진 yt-dlp가 받아 드라이브 「앳나운_영상/재활용_소스」에 업로드 → 코드방이 프레임 스캔·구간 컷 해서 B롤로 사용.
- 과거 영상 전체를 자산화하려면: 채널 업로드 목록을 `yt-dlp "https://www.youtube.com/@채널/videos" --flat-playlist -j` 로 뽑아 URL 리스트 확보 후 배치 다운(본진).

## 코드
- 다운: `yt-dlp` (본진). 코드방 반입: `curl -sL "https://drive.usercontent.google.com/download?id=<fid>&export=download&confirm=t" -o out.mp4`.
