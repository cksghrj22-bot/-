## 8. 긴 BGM 정중앙(이음매)이 무음 — 짧은 편만 아웃트로 BGM FAIL (2026-07-20)
- **증상**: 같은 긴 BGM으로 11(45초 나레이션)은 통과, 13(29초 나레이션)은 아웃트로 RMS **-50~-51dB**로 두 번 연속 FAIL. B롤 스타트를 바꿔도(오디오 무관) 동일. "플레이크"로 오해했으나 **결정적**.

## 11. 나레이션 영상 끝 음성 잘림 / 자막 어긋남 — TTS 스팬 불신·mux 트림 (2026-07-23 재아 숏)
- **증상**: 재아 숏에서 ①마지막 음성이 잘림 ②자막이 음성과 어긋남 ③mux 출력 오디오(40.1s)가 나레이션(43s)보다 짧게 잘림.
- **진짜 원인 3중**:
  - (a) `tts.build_narration_single`가 돌려준 줄 스팬이 **실제 오디오와 어긋남**(끝에 무음 꼬리 + 타이밍 드리프트). 스팬 끝(43.08s)과 실제 발화 끝(STT 42.48s)이 다름.
  - (b) 영상 세그를 스팬 duration으로 이어붙였는데 스팬이 비연속(겹침/틈)이라 **영상 길이≠나레이션 길이** → 끝 음성 잘림(v1) 또는 뒤 무음(v2).
  - (c) mux의 `loudnorm` + `amix duration=first`가 오디오를 **~3초 잘라먹음**.
- **해결(박제 — 나레이션 영상 철칙)**:
  1. TTS 나레이션은 **앞뒤 무음 트림**(`silenceremove`) 후 쓴다.
  2. 자막·컷 타이밍은 **build_narration 스팬을 믿지 말고 실제 오디오를 STT(shorts/stt.py, denoise=False)로 재측정**해 얻는다(롱폼과 동일 원리).
  3. 영상 세그는 **다음 줄 시작~(마지막은 나레이션 끝)까지 연속**으로 이어 영상 길이=나레이션 길이.
  4. mux는 `loudnorm` 대신 **`dynaudnorm`**(길이 불변). 렌더 후 **`ffprobe stream=duration`으로 오디오==비디오 검증** 필수.
  5. 아웃트로 "SNS에 일기를 쓰고 있어요"는 둘째 줄기 숏에도 넣는다(마지막 2.6s 하단·박스없음).
- **교훈**: 나레이션 싱크는 **실측(STT)+길이검증**이 정답. TTS 라이브러리 스팬·loudnorm은 조용히 어긋난다. 6·8·10번과 같은 '길이 정렬' 계열.

## 10. 아웃트로 0.1초 잘림 FAIL — 프레임 양자화 (2026-07-23 중단발펌 숏)
- **증상**: 새 숏 렌더가 `아웃트로 잘림 (끝 27.8s > 영상 27.7s)`로 7규격 FAIL. 눈으로는 멀쩡.

## 9. 자막이 화면 우상단에 붙음 — ASS Style 필드 하나 누락 (2026-07-23 러닝롱폼 실사고)
- **증상**: 롱폼 자막이 하단중앙이 아니라 **화면 우상단**에 뜸. 텍스트·폰트·박스는 정상이라 프레임 안 뽑아보면 못 잡음(형은 완성본을 못 봤어서 그동안 방치됨).
- **진짜 원인**: `[V4+ Styles]` 포맷은 필드 **15개**(…BackColour, Bold, BorderStyle, Outline, Shadow, **Alignment**, MarginL, MarginR, MarginV, Encoding). 그런데 Style 값에 `1,3,0,2,120,120,90,1`처럼 **꼬리값이 8개뿐** → **Alignment 필드가 통째로 누락** → libass가 나머지를 한 칸씩 당겨 읽어(MarginL을 Alignment로…) 정렬이 깨지고 우상단으로 감.
- **검증**: 단색 배경 + 1줄 테스트 ASS를 `-vf subtitles=`로 **1프레임만 구워** 위치 확인(대용량 렌더 전 2초 검증). 고친 값 `-1,3,6,0,2,140,140,70,1` → 하단중앙 정상.
- **수정(박제)**: Style 꼬리값은 **정확히 9개**(Bold,BorderStyle,Outline,Shadow,**Alignment**,MarginL,MarginR,MarginV,Encoding). 하단중앙 자막 = Alignment **2**. 값 개수를 셀 것.
- **교훈**: 5번(폰트)과 같은 부류 — **자막은 실제 구운 픽셀(위치·글자)로 검증**한다. ASS Style 필드는 개수가 하나만 어긋나도 조용히 밀려 읽혀 위치가 튄다. 큰 렌더 돌리기 전 **1프레임 위치 QC** 필수.

## 2026-07-23 #12 — 무음트림이 마지막 단어를 잘라먹음 (재아 숏 "되려고요"→"되려")
- **증상**: 나레이션 끝단어 "되려고요"가 "되려"에서 뚝 끊김(형: "말꼬리 흐리면 우유부단해 보임"). 길이는 맞아서(video≈audio) **길이검증만으로는 못 잡음** — 콘텐츠(단어)가 빠진 것.
- **진짜 원인**: `sync_pipeline`의 끝 무음트림 `silenceremove(-50dB)`이 종결어미 "요"의 **여린 tail(-36~-43dB)** 을 무음으로 오인해 잘라냄. raw 실제 말 41.5s인데 clean이 **41.25s로 절단** → 자막·오디오 둘 다 "고요" 소실.
- **검증법(박제)**: 길이 일치 ≠ 안 잘림. **끝 0.5s를 0.1s 단위 dB 스캔**해 종결어미 tail이 살아있는지 확인. raw vs clean 끝 파형 비교.
- **수정**: 끝트림 제거(시작만 트림, 형 나레이션은 앞 무음도 없음). raw를 **단어 완전착지+0.15 beat 지점(41.65s)까지 -t** 로 잘라 clean 생성. 마지막 자막 스팬 end도 단어 끝(~41.5)까지 연장. speechnorm이 끝단어를 -25dB로 올려 **딱 부러지게 착지**.
- **교훈**: TTS 종결어미(요/다/죠)는 여려서 무음트림에 잘 먹힌다. **끝단어는 트림 금지·하드컷(-t)으로 보존**. 검사는 항상 '끝 단어 dB 스캔'을 길이검증과 **함께**.
