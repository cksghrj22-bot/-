# 🤝 A·B·C 3방 협업 규약 (이찬호 2026-07-21 · 박제)

> 세 방이 **같은 레포(main) 커밋으로 소통**하며 함께 간다. 같은 일을 두 방이 동시에 하지 않는다(단일 줄기).
> 각 방은 자기 레인을 지키고, 겹칠 것 같으면 `knowledge/방_공유_작업로그.md` 맨 위에 한 줄 남겨 조율한다.

## 🔄 공통 동기화 (모든 방 필수)
- **응답(작업) 시작 전**: `git pull --rebase origin main`
- **커밋할 때마다**: `git pull --rebase origin main && git push origin HEAD:main` (거부되면 다시 pull 후 push, 성공까지)
- **충돌**: 최신(다른 방) 살리고 내 변경 재적용 → 테스트·`python3 -m shorts.spec` 통과 후 push.
- **큰 작업/인계**: `knowledge/방_공유_작업로그.md` 맨 위에 한 줄(누가·뭘·다음 방 할 일).
- **커밋 = 소통**: 별도 통보 없이, 커밋이 곧 공유. 다른 방은 pull로 최신 받는다.

## 🏠 방별 역할 (레인)
### A방 — 렌더·B롤 배관
- 렌더 파이프라인·오버레이(사진팝업 overlay.py)·B롤 카탈로그·교육자료 가공 등.
- B방이 준 대본·매칭을 받아 렌더/배관 이슈를 잡는다.

### B방 — 콘텐츠 생성·규격
- 대본 생성(차노보이스)·한/영 자막·상황판·B롤 주제매칭·게이트/규격 코드(verify_render).
- 형님 구술 → 대본 → 렌더 → 시안 → 드라이브 링크.

### C방 — 🆕 비서업무 (Assistant)
- **콘텐츠 코드는 만지지 않는다**(A/B 담당). 비서·조율·행정 전담.
- 업무:
  1. **상황판 리마인더 관리** — 형님이 오늘 처리할 요청·찍을 B롤·발행 대기를 매일 추려 띄운다.
  2. **율이 채점판 수거 추적** — 위원 12명 제출 현황 갱신 + 미제출 독려 문구 준비.
  3. **발행 큐 관리** — 블로그 9편·스레드 8편·유튜브 예약 상태 추적(발행은 승인 후).
  4. **형님 요청 취합** — 3종(①승인 yes/no ②로그인 필요한 값 ③복붙 한 줄)만 모아 한 번에.
  5. **노션 상태판·허브 동기화**(본진 자동화 페이지는 건드리지 않음).
  6. **스케줄/후속 알림** — 예약 생성은 본진 규약 존중(트리거 남발 금지).
- C방도 결과·현황은 **커밋으로 박제**(비서 로그: `knowledge/비서_로그.md` 권장).
