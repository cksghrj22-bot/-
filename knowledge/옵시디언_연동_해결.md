# 옵시디언(iCloud 볼트) 연동 — 근본원인과 해법 (2026-07-16 박제)

## 증상
`python3 -m pipeline check`가 「옵시디언 미연동」. 앱 OAuth(REST)로 볼트 폴더를 조회하면 **파일 0개**로 나옴.

## 근본원인 (확정)
- `secrets/gdrive.json`의 토큰 스코프 = `drive.file` + `youtube.upload` (tokeninfo로 확인).
- **`drive.file`은 "이 앱이 만든 파일"만 접근** → 브라우저에서 만든 옵시디언 볼트는 구조적으로 못 읽음(그래서 0개).
- 스코프를 넓히려 해도 **구글 기기 인증(Device Flow)은 `drive.readonly`를 거부**함 (`invalid_scope: Invalid device flow scope`). 즉 **앱 OAuth로는 영원히 볼트를 못 읽는다.**

## 해법 (이것만 됨)
볼트는 **사용자 스코프 MCP 구글드라이브 커넥터**로만 읽힌다.
- ✅ `read_file_content(fileId)` — **작동함** (자연어 표현으로 노트 본문 반환)
- ❌ `download_file_content(fileId)` — 이 세션에서 권한 스트림이 계속 닫힘. 쓰지 말 것.
- ✅ `search_files(parentId = '<폴더ID>')` — 폴더 순회/파일ID 수집용, 작동함.

### 절차 (새 세션에서 볼트 인덱싱하기)
1. `search_files`로 볼트 폴더를 재귀 순회해 `.md` 파일ID 수집.
2. 각 파일ID를 `read_file_content`로 읽어 로컬 미러 폴더에 저장(구조 유지).
3. `python3 -m pipeline add-vault <로컬미러경로>` → `pipeline check`에 「옵시디언 N개 포함」 뜨면 성공.

## 주의
- 이 세션은 MCP 커넥터가 30~60초마다 재접속을 반복해 read가 간헐 실패 → 실패하면 그 파일만 재시도.
- 볼트의 **개인 기록(일기 등)은 공개 리포지토리에 커밋 금지.** 인덱스(`data/`)는 gitignore라 안전(세션마다 재구축).
- 본진(맥) iCloud 정본에서 `pipeline add-vault "<iCloud 볼트경로>"`를 돌리는 게 가장 완전한 인덱싱(코드방은 미러 경유 보강).
