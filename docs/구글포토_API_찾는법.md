# 📸 Google Photos API 찾는 법 (집에서 할 일) — 2026-07-20

> 목표: A방이 Google Photos 「B롤」 앨범을 읽어 B롤로 쓰게 하기.
> ⚠️ 먼저 현실 체크부터 읽고 시작하세요. 헛수고 방지.

## ⚠️ 0. 먼저 알 것 — Google이 Photos API를 조였습니다 (2025.3)
- 예전엔 앱이 `photoslibrary.readonly`로 **내 사진 전체·앨범을 읽을 수** 있었는데, **2025년 3월부터 막혔어요.**
- 지금 third-party 앱이 할 수 있는 건 둘 뿐:
  1. **앱이 직접 올린 사진만** 접근 (내가 폰으로 찍은 기존 영상은 못 읽음)
  2. **Picker API** — 쓸 때마다 **내가 직접 사진을 골라야** 함 (자동 아님)
- 즉 **"B롤 앨범을 앱이 알아서 매일 읽기" = 지금은 사실상 어려움.**

### 👉 그래서 현실적 결론
**Photos API에 목매지 말고, 진짜 자동은 "Photos→Drive" 경로가 나아요.** (아래 3번)
그래도 시도해보고 싶으면 아래 1~2를 집에서 하면 됩니다.

---

## 1. (원해서 시도한다면) Google Cloud Console 세팅
> 유튜브·드라이브 API 이미 쓰니까 **같은 프로젝트 재사용** 가능.
1. `console.cloud.google.com` 접속 (같은 구글 계정)
2. 상단 프로젝트 선택 → 기존 프로젝트(유튜브/드라이브 쓰던 것) 고르기
3. 왼쪽 **API 및 서비스 → 라이브러리** → 검색창에 **`Photos Library API`** → 클릭 → **사용 설정**
4. **API 및 서비스 → OAuth 동의 화면**
   - 이미 설정돼 있으면 → **범위(스코프) 추가**에서 `photoslibrary.readonly` 찾아 추가
   - **테스트 사용자**에 본인 이메일 추가 (안 하면 인증 거부됨)
5. **API 및 서비스 → 사용자 인증 정보 → + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
   - 애플리케이션 유형: **데스크톱 앱**
   - 만들면 **JSON 다운로드** 버튼 → 그 파일 받기 (`client_secret_….json`)

## 2. 그 파일을 나한테 넘기기
- 받은 `client_secret_….json` 내용을 채팅에 붙여넣거나, 드라이브에 올려주세요.
- 내가 `secrets/gphotos_client.json`에 저장하고(gitignore·비밀), 기기 인증 플로우(gdrive처럼 링크+코드)를 붙일게요.
- 인증 끝나면 **되는 범위 안에서** 앨범/사진을 읽어봅니다. (막혀 있으면 3번으로 전환)

---

## 3. ✅ 진짜 자동 대안 — Photos → Drive (권장·안 막힘)
Photos API가 막혀도 이건 확실해요:
- **iPhone**: Google Photos에서 B롤 클립 여러 개 선택 → **공유/저장 → Google Drive → 「쇼츠_배경클립_넣는곳」**
- 또는 촬영 직후 **사진앱 → 공유 → Google Drive** 로 바로 (Photos 거치지 않고)
- Drive에 들어오면 **A방이 매시 스캔해서 B롤 카탈로그에 자동 등록.** 이게 지금 실제로 작동하는 파이프라인.

## 요약 (집에서 순서)
1. **먼저 3번(Drive 경로)** 로 B롤 몇 개 올려보기 → 바로 됨, 이게 본선.
2. 여유 되면 1~2번(Photos API client_secret) 발급해서 나한테 주기 → 되는지 내가 실험.
3. Photos API가 막혀 있으면 미련 없이 Drive 경로로 확정.

> 결론: **B롤 자동화의 실질 해답은 Drive 폴더.** Photos API는 "되면 보너스"로 접근.
