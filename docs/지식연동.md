# 지식 연동 가이드 — 모든 지식을 하나의 검색망으로

파이프라인의 목적: 잘 만드는 게 아니라 **연동·조합**. 모든 지식이 한 인덱스에서 검색되어야 조합 엔진(prompts/01)이 돌아간다.

## 연동 현황

| 원천 | 상태 | 연결 방법 |
|---|---|---|
| 노션 (허브·아이디어DB·자산지도·블로그) | ✅ 연동 중 | Claude 세션이 직접 읽음. 핵심 문서는 `knowledge/`에 사본 유지 |
| 이 리포지토리 (`knowledge/`, `content/`) | ✅ 인덱싱 | `python3 -m pipeline add knowledge && python3 -m pipeline add content` |
| **Obsidian vault** | 🔧 Mac에서 1줄 | vault는 마크다운 폴더다: `python3 -m pipeline add ~/경로/내Vault` 끝. 새 노트가 쌓이면 다시 실행 (중복은 자동 제거) |
| 구글드라이브 (사진·영상 원본) | 링크 참조 | 파일 자체는 인덱싱하지 않음 — 자산 지도(`knowledge/키워드_지도.md`)가 파일명↔키워드 매핑을 담당 |
| 유튜브/인스타 조회수 | ⏳ 키 대기 | `secrets/`에 키 배치 후 반응 데이터 → 조합 엔진 입력으로 (다음 단계) |

## 사용 예

```bash
# Obsidian vault까지 합쳐서 전체 지식 검색
python3 -m pipeline add ~/Obsidian/차노vault
python3 -m pipeline search "도제식"        # → 씨앗·블로그·옵시디언 노트가 한 번에
```

## 원칙

- **정본은 노션, 사본은 여기.** `knowledge/` 파일 상단에 항상 정본 링크를 남긴다.
- 파일 원본(영상·사진)은 드라이브, 읽는 지식은 인덱스 — 허브의 저장 규칙과 동일.
- 새 지식이 생기면: 노션에 정본 → `knowledge/`에 요약 사본 → `pipeline add` → 조합 엔진의 재료가 된다.
